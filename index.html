Overview

What is Llama Recipe Manager?

A native desktop application that lets you save, organize, and launch llama-server (llama.cpp) configurations as named recipes.

Instead of remembering long CLI flags or editing shell scripts every time you want to switch models, you create recipes once and launch them with one click. The app manages the server process, shows live metrics, and keeps your data local.

Built with Tauri 2, Rust, and Svelte 5 — it's fast, lightweight, and secure.

Recipe-First Workflow

Name your configurations, set the flags once, and launch with a single click.

Live Server Metrics

Monitor GPU usage, memory, tokens/sec, and context utilization in real time.

Process Management

Start and stop llama-server from the UI with full log visibility and status indicators.

Local & Private

No accounts, no telemetry, no network calls. Your data stays in a local SQLite database.

Features

Everything you need to manage llama-server

Designed for developers and AI enthusiasts who work with multiple GGUF models daily.

Named Recipes

Save each llama-server invocation as a named recipe with all flags. Switch between Mistral, Llama, CodeLlama, and more with one click.

Live Metrics Dashboard

Real-time GPU utilization, memory consumption, tokens-per-second, and context usage displayed alongside your running recipe.

Safety by Default

A flag deny-list blocks dangerous or conflicting arguments. Model paths are confined to your configured directory. Inputs are validated.

Local-First Storage

No accounts, no telemetry, no network calls. Every recipe lives in a SQLite database on your machine — private and yours alone.

Process Management

Start and stop llama-server from within the app. Full logs, status indicators, and automatic cleanup when you quit.

Cross-Platform

Available on macOS (.dmg), Linux (.deb, .rpm, .AppImage, AUR), and Windows (.msi, .exe). Same experience everywhere.
